# 单一职责原则(Single Responsibility Principle)

一个类只负责一项职责，即一个类应该只有一个引起它变化的原因。

## 原则说明

当一个类承担过多职责时，这些职责之间可能产生耦合，导致修改一个职责时会影响到其他职责。单一职责原则要求每个类只专注做一件事。

## 反面示例

```java
// 违反单一职责：一个类同时负责用户信息存储、验证和打印
public class User {
    private String name;
    private String email;

    // 存储职责
    public void save() { /* 保存到数据库 */ }

    // 验证职责
    public void validate() { /* 验证数据 */ }

    // 打印职责
    public void print() { /* 打印用户信息 */ }
}
```

## 正面示例

```java
// 职责分离后
public class User {
    private String name;
    private String email;
    // 只负责数据存储
}

public class UserValidator {
    public boolean validate(User user) { /* 验证逻辑 */ }
}

public class UserPrinter {
    public void print(User user) { /* 打印逻辑 */ }
}
```

## 优点

- 降低类的复杂度
- 提高代码可读性和可维护性
- 减少修改时对其他功能的影响
