# 迪米特法则(Least Knowledge Principle)

每个对象尽量减少对其他对象的了解，降低类之间的耦合。

## 原则说明

迪米特法则又称为最少知识原则，要求一个类只与直接朋友通信，不要和陌生人说话。直接朋友包括：
- 当前对象本身
- 以参数形式传入的对象
- 当前对象创建的对象
- 当前对象的成员对象

## 反面示例

```java
public class Teacher {
    public void command(GroupLeader leader) {
        // Teacher直接了解Student的细节
        List<Student> students = new ArrayList<>();
        for (int i = 0; i < 10; i++) {
            students.add(new Student());
        }
        leader.countStudents(students);
    }
}

public class GroupLeader {
    public void countStudents(List<Student> students) {
        System.out.println("学生数量: " + students.size());
    }
}
```

## 正面示例

```java
public class Teacher {
    public void command(GroupLeader leader) {
        // 只需要通知GroupLeader，不需要了解Student
        leader.countStudents();
    }
}

public class GroupLeader {
    private List<Student> students = new ArrayList<>();

    public GroupLeader(List<Student> students) {
        this.students = students;
    }

    public void countStudents() {
        System.out.println("学生数量: " + students.size());
    }
}
```

## 优点

- 降低类之间的耦合
- 提高类的复用性
- 减少类的复杂性
