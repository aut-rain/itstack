# 依赖倒置原则(Dependency Inversion Principle)

程序要依赖于抽象接口，不要依赖于具体实现，对抽象进行编程，降低耦合。

## 原则说明

依赖倒置原则包含两层含义：
1. 高层模块不应依赖低层模块，两者都应依赖抽象
2. 抽象不应依赖细节，细节应依赖抽象

## 反面示例

```java
// 违反依赖倒置：高层直接依赖低层实现
public class OrderService {
    private MySqlDatabase database = new MySqlDatabase();

    public void save(Order order) {
        database.save(order);
    }
}

// 如果要换数据库，需要修改OrderService
public class OracleDatabase {
    public void save(Order order) { /* ... */ }
}
```

## 正面示例

```java
// 定义抽象接口
public interface IOrderRepository {
    void save(Order order);
    void delete(Long id);
}

// 低层实现
public class MySqlRepository implements IOrderRepository {
    public void save(Order order) { /* MySQL保存 */ }
    public void delete(Long id) { /* MySQL删除 */ }
}

public class OracleRepository implements IOrderRepository {
    public void save(Order order) { /* Oracle保存 */ }
    public void delete(Long id) { /* Oracle删除 */ }
}

// 高层依赖抽象接口
public class OrderService {
    private IOrderRepository repository;

    // 通过构造函数注入，具体实现可替换
    public OrderService(IOrderRepository repository) {
        this.repository = repository;
    }

    public void save(Order order) {
        repository.save(order);
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        IOrderRepository repo = new MySqlRepository(); // 可替换为OracleRepository
        OrderService service = new OrderService(repo);
    }
}
```

## 优点

- 降低类之间的耦合
- 提高代码的可扩展性
- 便于单元测试（可注入Mock对象）
