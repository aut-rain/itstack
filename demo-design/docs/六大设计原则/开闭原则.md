# 开闭原则(Open Closed Principle)

软件实体应该对扩展开放，对修改关闭。

## 原则说明

开闭原则要求软件实体（类、模块、函数等）应该能够扩展新功能而不修改已有的代码。这可以通过抽象和多态来实现。

## 反面示例

```java
// 每次添加新的支付方式都需要修改PaymentService
public class PaymentService {
    public void pay(String type) {
        if ("alipay".equals(type)) {
            // 支付宝支付
        } else if ("wechat".equals(type)) {
            // 微信支付
        } else if ("bankcard".equals(type)) {
            // 银行卡支付
        }
        // 新增支付方式需要修改这个方法
    }
}
```

## 正面示例

```java
// 定义支付接口
public interface Payment {
    void pay();
}

// 具体支付实现
public class Alipay implements Payment {
    public void pay() { /* 支付宝支付 */ }
}

public class WechatPay implements Payment {
    public void pay() { /* 微信支付 */ }
}

public class BankCardPay implements Payment {
    public void pay() { /* 银行卡支付 */ }
}

// 使用方
public class PaymentService {
    public void pay(Payment payment) {
        payment.pay();
    }
}

// 新增支付方式只需添加新类，无需修改已有代码
public class UnionPay implements Payment {
    public void pay() { /* 银联支付 */ }
}
```

## 优点

- 提高代码的复用性
- 提高系统的可维护性
- 降低测试风险
